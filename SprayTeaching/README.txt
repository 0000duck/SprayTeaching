# 时间：2017-4-20 12:33:55
# 作者：xingshuang
# 修改内容：
1、完善RoboDK类中的相关功能；
	1）在启动软件的时候，若RoboDK软件未启动，则需要在启动自己之前启动RoboDK软件，这就导致自己启动延迟较大，影响使用；
	解决办法：创建新线程，在该线程中初始化，完成构造函数的功能，这样就可以在后台中对RoboDK软件进行启动，而不影响使用；
	2）在启动RoboDK软件会出现因路径不对而启动不了报错的问题；
	解决办法：在初始化线程中采用try-catch语句，在出错后不启动更新机器人参数的线程，并提示用户设置正确路径，重启软件；
	3）在自己软件正常运行，而RoboDK软件突然关闭而导致一些列的报错问题；
	解决办法：还是采用try-catch语句，在与RoboDK通信的地方进行监控，若出现错误则以日志的形式扔出，并用CheckRoboDK()函数进行检查连接或者重启RoboDK软件；
	4）在RoboDK软件因意外关闭后重启，会导致之前在RoboDK软件中选取的机器人对象无效或者不存在，从而引起报错；
	解决办法：在每次重启RoboDK软件之后，将this._rdkItemRobot=null，然后重新进行SelectRobot();
